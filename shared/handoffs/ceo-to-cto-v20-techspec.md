## 任务

制定 v2.0 完整技术方案（Tech Spec），存入 `shared/context-bus/cto/TECH-SPEC-V2.0.md`

## 输入文档

## 交付要求

Tech Spec 必须包含以下章节：

### 1. 架构设计
- 整体架构图（文本描述）
- 前端架构（组件层级、状态管理、渲染管线）
- 与 v1.x 的关系（重写 or 渐进升级）

### 2. 组件拆分
- 核心模块列表及职责
- 模块间依赖关系
- 目录结构规划

### 3. 技术选型
- 框架、库的选择及理由
- 构建工具、部署方案
- 动画/特效方案

### 4. 数据模型
- 游戏状态数据结构（TypeScript interface）
- 存档格式
- 配置表结构（境界、角色、关卡、妖怪、法宝等）

### 5. API 设计
- 游戏引擎 API（核心方法）
- 事件系统
- 存档 API

### 6. 里程碑拆分
- 分 4-6 个里程碑
- 每个里程碑的范围、预计工时、交付物
- 明确 MVP 最小可玩范围

## 注意事项
- v2.0 是大版本重构，允许重写但要说明迁移策略
- 关注性能：大数字运算、动画流畅度、离线计算
- 保持单页应用、GitHub Pages 部署
- 参考 PRD 中的数值体系（14个境界、81难、指数增长）
