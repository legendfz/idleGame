# v1.1 推广计划 & 反馈渠道方案

## 交付文件
- `CMO/V1.1-PLAN.md`

## 摘要

### Part 1：v1.1 更新推广
- **策略**：把版本更新包装为小型发布事件
- **节奏**：预告期 7 天（剧透→投票→开发日志→倒计时）→ 发布日多渠道同步 → 发布后社区活动
- **拉回流失用户**：回归奖励 + Reddit 评论通知 + changelog 强调「你的反馈我们改了」
- **文案模板**：Reddit 英文 + 中文社区帖已备好

### Part 2：用户反馈渠道
- **6 个渠道**：GitHub Issues（P0）、Discord（P0）、QQ群（P0）、TapTap、Reddit、游戏内入口
- **Discord 架构**：7 个频道（公告/闲聊/Bug/建议/投票/成就/开发日志）
- **处理流程**：收集→分类→排序→24h响应→跟进→闭环
- **游戏内反馈入口**：需 CTO 配合（跳转 GitHub Issue + 社区链接）

### KPI
- v1.1 更新日 DAU 恢复 >80% 峰值
- Discord 首月 200+ 成员
- 反馈 24h 响应率 100%

## 跨部门依赖
- **CPO**：确认 v1.1 功能清单以校准推广内容
- **CTO**：游戏内反馈入口 + GitHub Issue 模板创建
