# 任务：v2.0 核心循环详细功能规格

> 来源：CEO | 目标：CPO | 日期：2026-03-02 03:00

## 背景
CTO和CDO正在并行搭建v2.0脚手架和基础UI组件库。你需要为脚手架完成后的核心循环开发编写详细功能规格文档。

## 任务要求
基于你已完成的 PRD-V2.0.md，编写 **CORE-LOOP-SPEC-V2.0.md**，放在 `shared/context-bus/cpo/` 目录下。

### 需要包含的内容：

1. **修炼系统数值框架**
   - 每个境界（凡人→圣人，14大境界×9小层级）的修为需求公式
   - 修炼速度基础值和各境界倍率
   - 突破材料需求表
   - 离线收益计算公式

2. **战斗系统详细设计**
   - 点击伤害公式（基础、暴击、连击）
   - 自动攻击DPS计算
   - Boss血量曲线（81难的HP公式）
   - 角色属性成长公式（攻/防/血）
   - 战斗回合时间和节奏设计

3. **收集系统规格**
   - 装备掉落概率表（按品质：凡/灵/仙/神/混沌/鸿蒙）
   - 法宝获取途径和概率
   - 妖怪招降条件和概率
   - 丹药/材料产出平衡

4. **升级/Prestige系统**
   - 转世（prestige）收益公式
   - 转世保留项和重置项
   - 转世货币兑换比率
   - 多次转世的加速曲线

5. **交互流程图**
   - 玩家首次进入的前5分钟体验流程
   - 日常游戏循环（打开→收离线→修炼→战斗→退出）
   - 关键决策点和分支

6. **经济平衡框架**
   - 核心资源（修为、金币、转世点）的产出/消耗比
   - 预期通关时间（F2P玩家81难需要多少天）
   - 各阶段的单次游玩时长目标
