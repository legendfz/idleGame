# v17.0「渐入佳境」— 渐进解锁 + Tab整合

## 优先级：P0（影响新用户留存）

## 问题
当前10个底部Tab从游戏开始全部可见，新玩家极易迷失。

## 任务要求

### 1. 渐进Tab解锁系统
在 `gameStore.ts` 中添加解锁逻辑，Tab根据玩家进度逐步开放：

| Tab | 解锁条件 | 理由 |
|-----|---------|------|
| ⚔️ 战斗 | 初始 | 核心循环 |
| 👤 队伍 | 初始 | 查看角色 |
| 🏔️ 旅途 | 初始 | 推图 |
| 🎒 背包 | 等级5 或 获得首件装备 | 有内容才展示 |
| 🏆 成就 | 等级10 | 有成就可领才展示 |
| 🏔️ 洞天 | 等级20 或 首次突破 | 中期系统 |
| 🗺️ 秘境 | 等级30 | 中期探索 |
| 💕 仙缘 | 等级40 | 后期系统 |
| 📊 统计 | 等级15 | 有数据积累 |
| ⚙️ 更多 | 初始 | 设置始终可用 |

### 2. 实现方式
- `gameStore` 新增 `getUnlockedTabs()` 方法，基于 `player.level`, `highestChapter`, 装备数量等判断
- `BottomNav` 组件只渲染已解锁的Tab
- 新Tab解锁时，显示一个简短的 Toast 通知（如 "🎉 新功能解锁：背包"）
- 已解锁Tab存入存档，避免降级时Tab消失

### 3. Tab整合（次要）
- 如果Tab超过6个，使用"更多"作为折叠入口，把 统计/洞天/秘境/仙缘 放在"更多"的子菜单中
- "更多"页面展示：设置 + 已解锁的次要系统入口

### 4. 解锁通知
- 添加 `unlockedTabs: string[]` 到存档
- 新解锁时 push 一条 FloatingText 或 Toast

## 技术约束
- 修改 `App.tsx` 的 TABS 和 BottomNav
- 修改 `gameStore.ts` 添加解锁逻辑
- 修改 `types.ts` 如需新字段
- **不要新建文件**，在现有文件中修改
- Build必须通过：`cd CTO/idle-game && npm run build`

## 验收标准
- 新游戏开始只看到 3-4 个Tab
- 升级到对应等级后新Tab出现
- 解锁时有通知
- 存档加载后解锁状态保持
- Build 零错误
