# v35.0 全面 QA 审计报告

**测试环境**：https://legendfz.github.io/idleGame/ (显示版本 v31.0)
**测试日期**：2026-03-04
**测试方式**：浏览器实测 + 源码审查

## 🔴 P0 致命（影响核心体验）

### 1. 成就等级类永远无法完成
**描述**：「修行有成」(等级50) 和「登峰造极」(等级100) 两个成就永远不会触发，即使等级已远超目标。
**复现**：打开成就页 → Lv.200+ 时仍显示未完成
**根因**（代码审查）：
- `src/data/achievements.ts` 定义 `id: 'ach_level_50'`, `conditionType: 'level'`
- `src/store/achievementStore.ts:checkAchievements()` 的 switch 无 `case 'level'`（注释说"externally via updateProgress"）
- `src/App.tsx:119-120` 调用 `updateProgress('monkey_awaken', level)` 和 `updateProgress('level_100', level)` — ID 不匹配（应为 `'ach_level_50'` 和 `'ach_level_100'`）
- 结果：`ach_level_50` 从未被调用，`ach_level_100` 用错误 ID 调用
**修复建议**：App.tsx 中改为 `updateProgress('ach_level_50', level)` + `updateProgress('ach_level_100', level)`

### 2. 新手引导在高等级仍弹出（已知 bug）
**描述**：Lv.194 进入游戏时弹出「步骤 1/5: 开始修炼」
**复现**：任何高等级存档刷新页面
**根因**（代码审查）：
- `TutorialOverlay.tsx:23` 有 `level > 5` 守卫，应该隐藏
- 但 `gameStore.ts:973` 迁移逻辑 `tutorialStep ?? (level > 5 ? 6 : 1)` — `??` 只在 undefined/null 时生效
- 如果老存档中 `tutorialStep` 已存在为 `1`（初始值），迁移不触发
- 实测时 `level` 确实 > 5，但引导仍显示 → 说明 `tutorialDone` 为 false 且 `tutorialStep` 为 1，而 `level > 5` 的检查可能读到的是初始化时尚未加载的值
**修复建议**：在 TutorialOverlay 中直接从 store 读取 level，或在 load 时强制 `if (level > 5) tutorialDone = true`

### 3. 背包满 50 时掉落处理不透明
**描述**：背包 50/50 满，3 个装备栏全空，战斗持续掉落但新装备无处安放
**复现**：不装备任何东西 → 战斗持续 → 背包满后掉落去向不明（无提示）
**问题**：
- 用户没有任何"背包已满"的提示
- 没有自动装备更好装备的引导
- 统计显示"装备掉落 52"但背包只有 50 — 说明 2 件被静默丢弃
**修复建议**：背包满时显示提示气泡；或启用自动分解凡品

## 🟠 P1 重要

### 4. 成就完成率显示 1/15 (7%) 但多个应已达成
**描述**：Lv.200+, 击杀 1499, 装备 52 件，但成就仅 1/15 完成
**分析**：
- 「千妖斩」(击杀1000) 应已完成 → `kill_count` 在 checkAchievements 中有处理，可能 counter 未正确累计
- 「装备收藏家」(获得50件) 应已完成 → 同上
- 「富甲一方」(100万灵石) 已有 300万 → `gold_total` 类型在 switch 中存在，但 counter `totalGoldEarned` 可能未追踪历史总获取
**修复建议**：审查所有 achievement counter 的累加逻辑是否正确

### 5. 洞天·聚灵阵 功能描述与 PRD 不一致
**描述**：代码中聚灵阵 = "攻击力加成"(`produceType: 'attack'`)，PRD 定义为"离线收益加成"
**位置**：`src/engine/sanctuary.ts:22`
**影响**：聚灵阵功能完全偏离设计意图
**修复建议**：确认最终设计意图。如保留攻击加成需更新 PRD；如恢复离线收益需改代码

### 6. 仙缘 NPC 阵容与 PRD 不一致
**描述**：
- 实际：观音/太上老君/二郎神/哪吒/玉兔/龙女
- PRD v13.0：观音/太上老君/嫦娥/哪吒/二郎神/铁扇公主
- 玉兔和龙女替换了嫦娥和铁扇公主
**影响**：对应的 buff/专属神通也可能不同
**修复建议**：统一 PRD 与代码（建议以代码为准，更新 PRD）

### 7. 仙缘赠礼仅一种（100 灵石）
**描述**：PRD 定义三档赠礼（金币 5000/+3、灵石 10/+8、专属礼物/+15），实际只有 100 灵石一档
**影响**：好感度获取途径单一，节奏与 PRD 预期不同
**修复建议**：补充多档赠礼，或更新 PRD 为简化版

### 8. 秘境第 4 难度命名不一致
**描述**：代码显示"圣境"，PRD v13.0 定义"神境"
**影响**：仅命名差异，功能无影响
**修复建议**：统一命名

### 9. 品质体系简化 — 仅见凡品(○)/灵品(●)/仙品(◆)
**描述**：PRD 定义 6 品质（凡品/良品/上品/极品/仙品/鸿蒙），实际只观察到 3 级
**影响**：可能只是前期掉落范围，但需确认完整品质体系是否实现
**修复建议**：验证后期是否出现更高品质

## 🟡 P2 改进

### 10. 版本号显示 v31.0
**描述**：设置页显示 v31.0，但任务说是 v35.0 审计
**影响**：版本追踪混乱
**修复建议**：更新版本号

### 11. 秘境节点图标数与步数不一致
**描述**：进度显示"1/6"但节点图标显示 7 个（[箱][箱][泉][仙][刃][妖][箱]）
**影响**：轻微 UI 混淆
**修复建议**：节点数应与图标数一致

### 12. 战斗速度按钮缺少视觉反馈
**描述**：点击 1x→2x 速度切换正常工作，但无明显视觉区分（颜色/大小不变）
**影响**：玩家可能不确定当前速度档位
**修复建议**：高速模式时按钮高亮或变色

### 13. 背包排序无明显分类
**描述**：50 件装备平铺显示，仙品和凡品混在一起，没有默认按品质排序
**影响**：找装备困难
**修复建议**：默认按品质降序排列，或分组显示

### 14. 洞天建筑全 Lv.0 无引导
**描述**：Lv.200 时洞天全部 Lv.0，玩家可能不知道该升级
**影响**：功能发现率低
**修复建议**：增加红点提示或首次进入时的引导

### 15. 自动分解默认关闭
**描述**：设置中自动分解为"关闭"，导致凡品堆满背包
**影响**：新手体验差（背包满且不知道如何处理）
**修复建议**：背包超过 40 时提示开启自动分解；或默认开启凡品自动分解
