## 1. 系统概览

```
突破/转世/成就 → 获得天赋点 → 分配天赋路线 → 被动加成
战斗/秘境/成就 → 获得伙伴   → 升级+携带     → 被动加成
                                                    ↓
                              叠加进现有引擎公式（修炼/战斗/锻造/采集）
```

## 2. 天赋树系统

### 2.1 天赋点获取

| 来源 | 点数 | 说明 |
|------|------|------|
| 突破大境界 | +1 | 凡人→练气、练气→筑基…共 13 次（最多 13 点）|
| 转世 | +3 | 每次转世额外获得 |
| 成就 | +1 | 特定成就奖励（5 个成就各+1，共 5 点）|
| 里程碑「境界之路」💎 | +2 | 达到混元境界里程碑钻石级 |

**首次通关预估**：13（境界）+ 0（未转世）+ ~3（成就）= ~16 点
**多次转世后**：16 + 3×转世次数 + 剩余成就

### 2.2 天赋重置
- 转世时：天赋点**保留**，已分配的天赋**重置**（免费重新分配）
- 主动重置：消耗灵石×50，任意时刻重置全部天赋

### 2.6 天赋与引擎公式接入

```typescript
// 修炼速度
xiuwei_per_sec = base × realm × equip × team × buff × (1 + talent.cultivationBonus) × talent.cultivationMultiplier

// 攻击
attack = base_attack × (1 + equip% + talent.atkBonus) × talent.allStatsMultiplier

// 暴击率
crit_rate = base + equip + talent.critRateBonus

// 锻造成功率
forge_rate = recipe.rate + forgeLevelBonus + talent.forgeBonus + milestone.forgeBonus

// 采集
gather_time = base / (1 + talent.gatherSpeedBonus + charPassive)
gather_count = base × (1 + talent.gatherYieldBonus)
```

### 3.4 伙伴升级

每级消耗和效果提升：

```
升级费用:
  gold_cost(level) = 1000 × level^2
  material_cost(level) = T(ceil(level/3)) 材料 × level
  // Lv.1→2: 4,000 金 + T1×2
  // Lv.5→6: 36,000 金 + T2×6
  // Lv.9→10: 100,000 金 + T4×10

效果提升:
  bonus(level) = base × (1 + (level-1) × 0.15)
  // 每级+15%基础值（Lv.10 = 基础×2.35）
```

## 5. 技术实现建议

### 新增 Store
```
talentStore   — { points, allocated: Record<nodeId, boolean> }
petStore      — { owned: Record<petId, PetState>, activePetId }
```

### PetState 结构
```typescript
interface PetState {
  id: string;
  level: number;
  owned: boolean;
  obtainedAt: number;
}
```

### 存档扩展
`getFullState` 加入 talents + pets

### 事件监听
- `BREAKTHROUGH` → 天赋点+1
- `PRESTIGE` → 天赋点+3，天赋重置
- `BOSS_KILLED` → 判定伙伴掉落
- `DUNGEON_CLEAR` → 判定伙伴掉落
- `ACHIEVEMENT_UNLOCK` → 判定伙伴解锁 + 天赋点
