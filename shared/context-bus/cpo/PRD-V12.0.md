# PRD v12.0 —「仙途指引」

**版本**：12.0 | **作者**：CPO | **日期**：2026-03-03
**状态**：Draft → 待 CEO 审批
**依赖**：PRD-V2.0 ~ V11.0；v10.0 新手引导框架

---

## 1. 版本定位

v12.0 聚焦质量打磨，不新增玩法系统：

```
新手引导 → 降低流失，提升首日留存
设置面板 → 完善基础体验，支持存档管理
数据统计 → 可视化成就感，增强沉浸
```

> 注：v10.0 已设计了 5 步新手引导框架（tutorialStore）。本版本在其基础上扩展：增加系统解锁教程、高亮指引交互，以及完整的设置与统计面板。

### 3.3 存档导出/导入

#### 导出流程
1. 点击「导出存档」
2. 调用 `getFullState()` → JSON → base64 编码
3. 复制到剪贴板，显示 ✅ 已复制
4. 提示用户保存到安全位置

#### 导入流程
1. 点击「导入存档」
2. 弹出输入框，粘贴 base64 字符串
3. 解码 → JSON → 校验格式版本
4. 二次确认：「导入将覆盖当前存档，确认？」
5. 调用 `loadFullState()` → 刷新 UI

#### 校验规则
- 必须是合法 base64
- JSON 必须包含 `version` 字段
- 版本低于当前时触发迁移逻辑
- 格式错误时提示「存档格式无效」，不覆盖

### 3.4 重置存档

1. 点击「重置存档」
2. 第一次确认：「确定要重置所有数据？此操作不可撤销！」
3. 第二次确认：输入「重置」二字
4. 清除所有 store → 初始化默认值 → 刷新 UI

## 5. Store 设计

### 5.1 settingsStore

```typescript
interface SettingsState {
  animationEnabled: boolean;
  autoSaveInterval: 15 | 30 | 60 | 120;  // 秒
  battleLogDetail: 'simple' | 'detailed';
  soundEnabled: boolean;
  soundVolume: number;  // 0-100
}
```

### 5.2 statsStore（扩展现有统计字段）

```typescript
interface StatsState {
  // 时间
  totalCultivateTime: number;   // 秒
  totalOnlineTime: number;      // 秒

  // 战斗
  totalKills: number;
  maxDamage: number;
  bossKills: number;
  pvpWins: number;

  // 成长
  reincarnations: number;
  highestRealm: number;         // realmOrder
  breakthroughs: number;

  // 收集
  equipObtained: number;
  bestEquipQuality: number;     // quality tier 0-5
  totalForges: number;
  forgeSuccesses: number;

  // 灵兽
  maxPetLevel: number;

  // 经济
  totalGoldEarned: number;
  totalSpiritStones: number;
}
```

### 5.3 tutorialStore 扩展

在 v10.0 基础上新增：

```typescript
interface TutorialState {
  // ... v10.0 字段保留
  systemTutorials: Record<string, boolean>;  // 系统解锁教程已读标记
  // key: 'forge' | 'tower' | 'pet' | 'talent' | 'skill' | 'alliance' | 'pvp'
}
```

### 5.4 存档集成

`getFullState()` / `loadFullState()` 必须包含 settingsStore + statsStore + tutorialStore 扩展字段。

## 9. 技术注意事项

1. **引导遮罩**：纯文字游戏不用图形遮罩，用 emoji 箭头+高亮文字实现
2. **存档导出格式**：`btoa(JSON.stringify(getFullState()))` → 剪贴板
3. **存档导入校验**：try-catch base64 decode + JSON.parse + version 字段检查
4. **统计埋点**：在各引擎关键路径插入统计累加（battle.onKill、forge.onComplete 等）
5. **自动保存**：gameLoop tick 中按 `settingsStore.autoSaveInterval` 间隔调用 save
6. **性能**：动画关闭时跳过 UI 更新中的动画相关 setState
