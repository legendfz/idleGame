# PRD v9.0 —「悟道成仙」

**版本**：9.0 | **作者**：CPO | **日期**：2026-03-02
**状态**：Draft → 待 CEO 审批
**依赖**：PRD-V2.0 ~ V8.0（修炼/战斗/转世/通天塔等）

---

## 1. 系统概览

```
转世获得天赋点 → 天赋树（3条路线）→ 永久强化
击败Boss/通天塔 → 悟道值 → 神通技能升级
转世≥3次 → 解锁自动战斗策略 → 挂机优化
```

三系统为游戏中后期增加策略深度：天赋树提供转世回报感，神通技能丰富战斗维度，自动战斗策略提升挂机体验。

## 4. 自动战斗策略系统

### 4.1 解锁条件

- 转世次数 ≥ 3

### 4.2 三套预设策略

| 策略 | emoji | 攻击权重 | 防御权重 | 修炼权重 | 适用场景 |
|------|-------|---------|---------|---------|---------|
| 🗡️ 全力输出 | 🗡️ | ×1.5 | ×0.5 | ×0.8 | 快速推图/通天塔冲层 |
| ⚖️ 均衡发展 | ⚖️ | ×1.0 | ×1.0 | ×1.0 | 默认，无特殊加成 |
| 💤 挂机效率 | 💤 | ×0.7 | ×1.2 | ×1.5 | 长时间离线/修炼为主 |

### 4.3 权重应用

策略权重影响以下计算：

```
effective_atk = base_atk × strategy.atk_weight
effective_def = base_def × strategy.def_weight
effective_cultivate_speed = base_speed × strategy.cultivate_weight
```

- 权重只影响自动战斗/挂机期间
- 手动操作（点击攻击、手动挑战）不受策略影响
- 策略可随时切换，无冷却

### 4.4 策略切换

- 在角色面板或设置中切换
- 切换立即生效
- 默认策略：⚖️ 均衡发展

## 5. Store 设计

### 5.1 talentStore

```typescript
interface TalentState {
  availablePoints: number;        // 可用天赋点
  totalEarned: number;            // 历史累计获得
  trees: {
    strength: number[];           // [0]=层1已点(0/1), [1]=层2...共8
    mind: number[];
    fortune: number[];
  };
  lastResetTime: number;          // 上次重置时间戳 ms
}
```

### 5.2 skillStore（神通）

```typescript
interface SkillState {
  skills: Record<string, {
    id: string;
    level: number;                // 1-5
    unlocked: boolean;
    cooldownUntil: number;        // 主动技能冷却截止时间戳
    activeUntil: number;          // 主动效果持续截止时间戳
  }>;
  wisdomPoints: number;           // 悟道值
}
```

### 5.3 strategyStore

```typescript
interface StrategyState {
  unlocked: boolean;              // 转世≥3解锁
  activeStrategy: 'offense' | 'balanced' | 'idle';
}
```

### 5.4 存档集成

`getFullState()` / `loadFullState()` 必须包含 talentStore + skillStore + strategyStore。

## 7. 系统联动

### 7.1 天赋 → 战斗/修炼/掉落
- 力道路线直接增强战斗输出（含通天塔）
- 心法路线增强修炼效率（含离线）
- 机缘路线增强掉落（含锻造成功率）

### 7.2 神通 → 多系统
- 被动加成常驻，影响所有对应计算
- 主动技能为在线冲刺手段

### 7.3 转世 → 天赋/策略
- 转世是天赋点的核心来源，鼓励多次转世
- 3次转世解锁策略系统，增强挂机体验

### 7.4 成就联动建议（v4.0）
- 「天赋初觉」— 点亮第 1 个天赋
- 「天赋大成」— 任一路线全部点亮
- 「三修圆满」— 三条路线全部点满
- 「神通初成」— 解锁第 4 个神通
- 「大圣归来」— 七十二变升至 Lv.5

### 7.5 每日任务联动建议
- 「触发 1 次神通」— 奖励悟道值 ×10

## 10. 技术注意事项

1. **天赋加成注入**：在 `formulas.ts` 增加天赋加成参数，与灵兽加成同层计算
2. **神通被动**：注册到 `eventBus` 或在 tick 中检查生效
3. **神通主动**：UI 触发 → store 更新 activeUntil/cooldownUntil → tick 中检查过期
4. **策略权重**：仅修改自动战斗 tick 的属性计算，不改基础 store 值
5. **存档兼容**：旧存档无 talent/skills/strategy 字段时初始化默认值
6. **天赋转世保留**：reincarnation 逻辑中不清除天赋/神通数据
