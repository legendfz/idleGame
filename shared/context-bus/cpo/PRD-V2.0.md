# PRD v2.0 — 西游记 Idle Game

> 版本：2.0 | 作者：CPO | 日期：2026-03-02
> 状态：Draft → 待 CEO 审批

## 1. 产品愿景

将经典西游记融入 Idle Game 玩法，玩家扮演取经团队，通过挂机修炼、点击战斗、收集法宝，重走九九八十一难的取经之路。

## 2. 核心玩法

### 2.1 Idle Loop（挂机循环）

```
修炼（自动产出修为） → 突破境界 → 解锁新能力 → 更快产出
         ↓
   取经进度推进 → 遭遇劫难 → 战斗/解谜 → 获得奖励
         ↓
   收集法宝/装备 → 强化角色 → 推进更远
```

- **修为**：核心资源，每秒自动产出，数值随境界指数增长
- **修炼速度**：base × 境界倍率 × 法宝加成 × 团队buff
- **突破**：积累足够修为后消耗资源突破到下一境界
- **挂机上限**：最多积累 24 小时离线收益，VIP 可扩展到 48 小时

### 2.2 点击机制

- **普通点击**：产出 10× 每秒修为量
- **暴击点击**：5% 概率触发 50× 产出，附金光特效
- **连击加速**：连续点击 10 次以上，触发 2× 速度 buff，持续 5 秒
- **Boss 战点击**：关卡 Boss 出现时，点击 = 攻击，需在限时内击败

### 2.3 离线收益

- 基础：离线时间 × 修炼速度 × 50%（离线效率折扣）
- 加成：孙悟空"定身法"技能 → 离线效率 +20%
- 回归奖励：离线 > 8 小时额外 bonus
- 保底：离线收益不低于在线 30 分钟等值

## 3. 西游记主题设定

### 3.1 角色体系

### 3.2 修炼境界

```
凡人 → 练气 → 筑基 → 金丹 → 元婴 → 化神 → 渡劫 → 地仙 → 天仙 → 金仙 → 太乙 → 大罗 → 混元 → 圣人
```

每个大境界 9 个小层级（对应九九之数），每次突破需要：
- 足够的修为值
- 对应的突破材料（丹药/天材地宝）
- 部分境界需通过试炼关卡

## 4. 核心系统设计

### 4.1 取经进度系统

- **总进度**：九九八十一难，线性推进
- **每一难**：包含 3-5 波小怪 + 1 Boss
- **难度曲线**：指数增长，每 9 难一个大台阶
- **通关条件**：击败 Boss（点击战斗 + 自动攻击混合）
- **星级评价**：每难 1-3 星，影响奖励质量
- **扫荡**：3 星通关后可扫荡，快速获取资源

### 4.2 装备/法宝系统

#### 装备槽位（每角色 6 个）

| 槽位 | 示例 |
|------|------|
| 武器 | 如意金箍棒、九齿钉耙、降妖宝杖 |
| 头饰 | 紧箍咒、凤翅紫金冠 |
| 衣服 | 锁子黄金甲、袈裟 |
| 饰品 | 定海神针（缩小版）、佛珠 |
| 坐骑 | 筋斗云、白龙马 |
| 法宝 | 紫金红葫芦、芭蕉扇、照妖镜 |

#### 品质等级（沿用 v1 体系）

凡品 → 灵品 → 仙品 → 神品 → 混沌 → 鸿蒙

#### 强化系统

- +1 ~ +10：100% 成功率递减（+10 为 30%）
- +11 ~ +15：需要保护材料，失败降级
- 精炼：同品质装备合成，提升基础属性

### 4.3 技能树

每个角色独立技能树，3 条路线：

**孙悟空示例：**
- 🔴 战斗路线：棒法精通 → 火眼金睛 → 大闹天宫
- 🔵 变化路线：七十二变 → 分身术 → 法天象地
- 🟡 修炼路线：仙丹体质 → 不死之身 → 斗战胜佛

技能点来源：升级 + 取经进度 + 成就

### 4.4 Prestige（转世系统）

- 通关第 81 难后解锁"如来点化"
- 转世重修：重置进度，保留部分永久加成
- 转世货币"佛缘"：用于购买永久升级
- 每次转世增加全局倍率

## 5. 货币与资源体系

| 货币/资源 | 获取方式 | 用途 |
|-----------|----------|------|
| 修为 | 挂机自动产出 | 突破境界 |
| 金币 | 击败妖怪、扫荡 | 购买道具、强化 |
| 灵石 | 关卡首通、成就、每日任务 | 高级抽取、加速 |
| 天材地宝 | Boss 掉落、炼制 | 突破材料、炼丹 |
| 仙丹 | 炼丹系统产出 | 临时 buff、突破辅助 |
| 佛缘 | 转世获得 | 永久升级 |

### 通胀控制
- 修为：指数增长需求，避免数值膨胀过快
- 金币：消耗口多（强化、炼丹、商店）
- 灵石：硬通货，产出稳定且有限

## 6. 里程碑关卡设计（九九八十一难）

| 章节 | 难数 | 主题 | 标志性 Boss |
|------|------|------|-------------|

- **每 9 难**：章节 Boss，需要特殊策略
- **隐藏难**：满足条件解锁番外关卡（如"大闹天宫"回忆关）
- **每日劫难**：随机生成，奖励丰厚

## 8. 用户故事与验收标准

### US-001：挂机修炼
> 作为玩家，我希望角色自动修炼产出修为，以便我不在时也有进展。

**验收标准：**
- [ ] 进入游戏后，修为每秒自动增加
- [ ] 界面显示每秒修为产出速度
- [ ] 修为数值实时更新（每秒刷新）
- [ ] 数值格式：< 10000 显示原数，≥ 10000 用 K/M/B/T 缩写

### US-002：境界突破
> 作为玩家，我希望积累修为后突破到更高境界，获得更强能力。

**验收标准：**
- [ ] 修为达标时显示"可突破"按钮
- [ ] 点击突破消耗修为，播放突破动画
- [ ] 突破后修炼速度倍率提升
- [ ] 境界名称显示在角色面板

### US-003：点击战斗
> 作为玩家，我希望通过点击攻击妖怪，在关卡中推进取经进度。

**验收标准：**
- [ ] 进入关卡后，屏幕显示妖怪
- [ ] 每次点击造成伤害，妖怪血条减少
- [ ] 角色自动攻击（DPS = 每秒自动伤害）
- [ ] 妖怪被击败后掉落金币和装备
- [ ] Boss 战有倒计时，超时则失败

### US-004：离线收益
> 作为玩家，我希望关闭游戏后再打开时获得离线期间的收益。

**验收标准：**
- [ ] 重新打开游戏显示离线收益面板
- [ ] 显示离线时长、获得修为、获得金币
- [ ] 离线收益 = 离线时长 × 修炼速度 × 50%
- [ ] 最多计算 24 小时
- [ ] 点击"领取"后资源入账

### US-005：装备获取与穿戴
> 作为玩家，我希望从战斗中获得装备并穿戴到角色身上。

**验收标准：**
- [ ] 击败妖怪有概率掉落装备
- [ ] 装备显示名称、品质颜色、属性数值
- [ ] 可将装备拖拽/点击装备到对应槽位
- [ ] 穿戴后角色属性实时更新
- [ ] 背包满时提示分解或丢弃

### US-006：装备强化
> 作为玩家，我希望消耗金币强化装备，提升其属性。

**验收标准：**
- [ ] 选择装备后显示强化按钮和成功率
- [ ] 消耗金币执行强化
- [ ] 成功：等级+1，属性提升，播放成功动画
- [ ] 失败：显示失败提示，+10 以下不降级
- [ ] 强化等级显示在装备名称上（如 +7 如意金箍棒）

### US-007：取经地图
> 作为玩家，我希望看到取经进度地图，了解自己走到了哪里。

**验收标准：**
- [ ] 主界面可切换到地图视图
- [ ] 显示 81 个节点的路线图
- [ ] 已通关节点亮起，当前节点高亮，未解锁节点灰暗
- [ ] 点击已通关节点可扫荡（3 星后）
- [ ] 点击当前节点进入战斗

### US-008：角色切换
> 作为玩家，我希望切换控制不同角色，体验不同玩法。

**验收标准：**
- [ ] 角色面板显示已解锁角色列表
- [ ] 点击角色可切换为主控
- [ ] 不同角色有不同被动技能生效
- [ ] 切换无冷却时间

## 9. 技术约束（给 CTO 参考）

- 前端：TypeScript + Vite（沿用 v1 架构）
- 存储：localStorage，单存档
- 大数字：使用 break_infinity.js 或类似库
- 帧率：游戏循环 60fps，UI 刷新可降至 10fps
- 离线计算：重新打开时一次性计算，不用 Service Worker
- 打包体积：< 2MB（不含图片资源）
