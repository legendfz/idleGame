**版本**：4.0 | **作者**：CPO | **日期**：2026-03-02

**依赖**：PRD-V2.0 / PRD-FORGE-SYSTEM(v3.0)

## 1. 系统概览

```
日常游玩 → 触发成就/完成任务/推进里程碑 → 获取奖励+永久加成 → 更高效游玩
```

三大留存子系统相互交织：
- **成就**：一次性目标，提供稀有奖励和称号
- **每日任务**：每日循环，保证日活参与度
- **里程碑**：长线追踪，提供永久 buff

## 6. 数值平衡分析

### 6.1 每日任务经济影响

| 指标 | 数值 |
|------|------|
| 每日任务平均金币产出 | ~15,000（5 条平均 + 全勤 10K）|
| 每日任务灵石产出 | ~25（任务 + 全勤 20）|
| 每日任务材料产出 | T1×5~10, T2×3~8 |
| 占中期每日总收入比例 | 金币 ~15%, 灵石 ~30% |

**设计意图**：每日任务是灵石主要来源（驱动每日登录），金币为辅助来源。

## 7. 技术实现建议

### 新增 Store
```
achievementStore — 成就状态 { [achId]: { unlocked, claimedAt } }
dailyQuestStore  — 每日任务 { quests[], progress{}, resetDate }
milestoneStore   — 里程碑 { [msId]: currentTier }
statsStore       — 统计追踪 { totalKills, totalForgeCount, ... }
```

### 存档扩展
`getFullState` 需加入：achievements, dailyQuests, milestones, stats

### 事件驱动
复用 eventBus 监听：
- `BREAKTHROUGH` → 检查修炼成就 + 境界里程碑
- `ENEMY_KILLED` → 击杀统计 + 战斗成就
- `FORGE_COMPLETE` → 锻造统计 + 锻造成就
- `GATHER_COMPLETE` → 采集统计 + 采集成就
- `STAGE_COMPLETE` → 探索成就 + 取经里程碑
- `DUNGEON_CLEAR` → 秘境里程碑
