**版本**：8.0 | **作者**：CPO | **日期**：2026-03-02

## 1. 系统概览

```
通天塔（无尽挑战） → 层数奖励 + 灵兽解锁
排行榜（本地NPC） → 每日结算排名奖励
灵兽系统（被动加成） → 提升战力 / 修炼 / 经济
```

三系统相互联动：通天塔提供灵兽获取渠道；排行榜以通天塔层数为核心榜单之一；灵兽加成反哺通天塔推进与修炼效率。

## 2. 通天塔系统

### 2.1 基本规则

| 属性 | 值 |
|------|-----|
| 层数上限 | ∞（无限） |
| 每层敌人数 | 1（普通）/ 1（精英/Boss） |
| 敌人属性缩放 | 基础 × 1.08^层 |
| 每日免费挑战次数 | 3 |
| 额外次数（灵石购买） | 最多 +3 次/日，费用递增 |
| 失败处理 | 停留当前层，不扣次数 |
| 扫荡 | 已通关层可一键扫荡 |

### 2.2 敌人类型

| 类型 | 出现条件 | 特殊机制 |
|------|---------|---------|
| 🟢 普通怪 | 默认每层 | 无特殊 |
| 🟡 精英怪 | 每 10 层（10/20/30…） | 1 个随机特殊技能 |
| 🔴 Boss | 每 50 层（50/100/150…） | 2 个特殊技能 + 限时 60s |

### 2.3 精英/Boss 特殊技能池

| 技能 | 效果 | emoji |
|------|------|-------|
| 铁壁 | 防御 ×3 | 🛡️ |
| 狂暴 | 攻击 ×2，防御 ×0.5 | 💢 |
| 回春 | 每 5s 回复 5% 最大 HP | 💚 |
| 分身 | 实际承受伤害 ×0.5（等效 HP 翻倍） | 👥 |
| 反伤 | 反弹 10% 受到的伤害 | 🔄 |
| 吸血 | 攻击时回复造成伤害的 15% | 🧛 |

精英怪随机 1 个，Boss 随机 2 个（不重复）。

### 2.4 敌人属性公式

```
base_hp   = 500
base_atk  = 30

floor_hp(n)  = base_hp × 1.08^n
floor_atk(n) = base_atk × 1.08^n

精英加成：hp ×3, atk ×2
Boss加成：hp ×10, atk ×5
```

### 2.6 扫荡机制

- 条件：已通关的层（不含当前未通关层）
- 操作：选择扫荡至第 N 层（1 ≤ N ≤ 已通关最高层）
- 消耗：1 次挑战次数
- 产出：累计 1~N 层的灵石+金币奖励（经验不给，里程碑不重复）
- 扫荡奖励系数：×0.5（防止无限刷）

### 2.7 额外挑战次数购买

| 购买次序 | 灵石费用 |
|---------|---------|
| 第 1 次 | 20 |
| 第 2 次 | 50 |
| 第 3 次 | 100 |

每日重置购买次数。

## 3. 排行榜系统

### 3.1 榜单类型

| 榜单 | 排序依据 | 数据来源 |
|------|---------|---------|
| 🏔️ 修为榜 | 总修为值 | playerStore.cultivationTotal |
| 🗼 通天塔榜 | 历史最高通关层数 | towerStore.highestFloor |
| ⚔️ 战力榜 | 综合战力值 | 计算值（见3.3） |

### 3.2 NPC 排行生成

复用现有 `leaderboard.ts` 的 NPC 生成逻辑，扩展为三榜：
- 每个榜单独立生成 19 个 NPC
- NPC 实力分布：玩家值的 30%~300%
- 玩家数据实时插入排序

### 3.3 战力计算公式

```
power = (atk × 2 + def × 1.5 + hp × 0.1) × realmMultiplier × petBonus
```

其中 `petBonus = 1 + 灵兽攻击加成%/100`（无灵兽时为 1）。

### 3.4 排名奖励（每日结算）

每个榜单独立结算：

| 排名 | 灵石 | 金币 | 额外 |
|------|------|------|------|
| 第 1 名 | 50 | 20,000 | 灵兽丹 ×1 |
| 第 2-3 名 | 30 | 12,000 | — |
| 第 4-10 名 | 15 | 6,000 | — |
| 第 11-20 名 | 5 | 2,000 | — |

三榜奖励可叠加。

| 来源 | 数量 |

- 同时只能携带 **1 只** 灵兽

- 未拥有的灵兽显示为灰色锁定状态

```typescript
interface TowerState {
  currentFloor: number;        // 当前挑战层（从1开始）

```

### 5.2 petStore

```typescript
interface Pet {
  id: string;           // 'qinglong' | 'baihu' | 'zhuque' | 'xuanwu' | 'qilin'

  emoji: string;
  level: number;        // 1-10
  bonusType: string;    // 'atk%' | 'def%' | 'cultivateSpeed%' | 'dropRate%' | 'spiritStone%'
  bonusBase: number;
  bonusPerLevel: number;
  owned: boolean;

interface PetState {
  pets: Record<string, Pet>;
  activePetId: string | null;
  switchCooldownUntil: number; // timestamp ms
  petPills: number;            // 灵兽丹数量
}
```

## 7. 系统联动

### 7.1 灵兽 → 通天塔
- 青龙/白虎加成直接影响通天塔战斗数值
- 朱雀不影响通天塔（塔内无修炼）
- 玄武增加通天塔掉落率
- 麒麟增加通天塔灵石奖励

### 7.2 通天塔 → 灵兽
- 通天塔里程碑是灵兽的核心获取渠道
- 里程碑也提供灵兽丹

### 7.3 通天塔 → 排行榜
- 通天塔最高层数直接作为排行数据

### 7.4 成就系统联动（v4.0）
建议新增成就：
- 「初登通天」— 通天塔 10 层
- 「塔中豪杰」— 通天塔 100 层
- 「万仙阵主」— 通天塔 300 层
- 「灵兽大师」— 拥有全部 5 只灵兽
- 「至尊灵兽」— 任一灵兽达到 Lv.10

### 7.5 每日任务联动（v4.0）
建议新增每日任务：
- 「通天塔挑战 3 层」— 奖励灵兽丹 ×1
- 「排行榜查看」— 奖励金币 ×2,000

## 8. 数据持久化

### 8.1 存档字段

新增字段加入 `getFullState()` / `loadFullState()`：

```typescript
// 必须在存档中包含
tower: TowerState;
pets: PetState;
leaderboard: LeaderboardState;  // 仅存 lastSettleDate + 玩家历史最佳
```

### 8.2 每日重置逻辑

在 `tick` 或 `gameLoop` 中检查日期变更：
1. 重置 `towerStore.dailyAttempts = 0`
2. 重置 `towerStore.dailyPurchased = 0`
3. 结算排行榜奖励
4. 重新生成 NPC 排行数据

## 10. 技术注意事项

1. **通天塔战斗**：复用现有 `battle.ts` 的 BattleCalc，新增特殊技能修饰器
2. **排行榜扩展**：现有 `leaderboard.ts` 仅单榜 NPC 生成，需扩展为三榜
3. **灵兽加成注入**：在 `formulas.ts` 的攻击/防御/修炼公式中增加 petBonus 参数
4. **存档兼容**：旧存档无 tower/pets 字段时，初始化默认值
5. **日期重置**：使用本地时区 `YYYY-MM-DD` 判断日期变更
