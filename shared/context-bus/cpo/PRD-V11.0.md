# PRD v11.0 —「仙盟争锋」

**版本**：11.0 | **作者**：CPO | **日期**：2026-03-03
**状态**：Draft → 待 CEO 审批
**依赖**：PRD-V2.0 ~ V10.0

> ⚠️ 本游戏为单机 idle game，所有"多人"交互使用模拟/NPC 数据实现。

---

## 1. 系统概览

```
仙盟（公会）→ 捐献/任务/等级加成 → 社交归属感
论道擂台（PvP）→ 异步镜像对战 → 竞技成就感
仙界盛会（限时活动）→ 竞速/共伐/寻宝 → 限时紧迫感
```

## 2. 仙盟系统

### 2.1 基本规则

| 属性 | 值 |
|------|-----|
| 人数上限 | 30（含盟主） |
| 创建费用 | 500 灵石 |
| 每人限加入 | 1 个仙盟 |
| 退出冷却 | 24 小时内不可再加入其他仙盟 |
| 解散条件 | 仅盟主可解散（成员≤1 时） |

### 2.2 权限层级

| 角色 | 人数 | 权限 |
|------|------|------|
| 🟡 盟主 | 1 | 全部管理 + 解散 + 任命长老 |
| 🔵 长老 | ≤3 | 审批入盟 + 踢人 + 仓库管理 |
| ⚪ 成员 | ≤26 | 捐献 + 领取 + 接任务 |

盟主可转让盟主身份给长老。盟主 72h 不上线自动转让给最高贡献长老。

### 2.3 仙盟仓库

#### 捐献

| 可捐物品 | 获得贡献点 |
|---------|-----------|
| 金币 ×10,000 | 10 贡献点 |
| 灵石 ×5 | 15 贡献点 |
| T1 材料 ×10 | 5 贡献点 |
| T2 材料 ×5 | 10 贡献点 |
| T3 材料 ×3 | 20 贡献点 |

每日捐献上限：100 贡献点/人

#### 领取（消耗个人贡献点）

| 物品 | 费用(贡献点) | 每日限量 |
|------|------------|---------|
| 小修为丹 ×3 | 20 | 3 次 |
| 强化符 ×2 | 30 | 2 次 |
| 灵兽丹 ×1 | 40 | 1 次 |
| 悟道丹(悟道值+50) | 50 | 1 次 |
| 天赋果(天赋点+1) | 200 | 每周 1 次 |

### 2.4 仙盟任务

每日刷新 3 个任务，全盟成员共同完成：

| 任务类型 | 示例 | 目标值 | 奖励(每人) |
|---------|------|--------|-----------|
| 修炼类 | 全盟累计修炼 1,000 次 | 1,000 | 30 贡献点 + 金币 5,000 |
| 战斗类 | 全盟累计击败 500 敌人 | 500 | 30 贡献点 + 灵石 5 |
| 锻造类 | 全盟累计锻造 50 次 | 50 | 30 贡献点 + 强化符 ×1 |
| 通天塔类 | 全盟累计通关 100 层 | 100 | 40 贡献点 + 灵兽丹 ×1 |
| 捐献类 | 全盟累计捐献 300 贡献点 | 300 | 20 贡献点 + 金币 8,000 |

任务池随机抽 3 个（不重复类型）。全部完成额外奖励：50 贡献点。

> **NPC 模拟**：其他 NPC 成员每日自动贡献随机进度（总目标的 40%~70%），玩家只需完成剩余部分。

## 3. 论道擂台（PvP）

### 3.1 基本规则

| 属性 | 值 |
|------|-----|
| 每日免费次数 | 5 |
| 额外次数 | 灵石购买，最多 +3（20/40/80） |
| 匹配范围 | 战力 ±20% |
| 战斗方式 | 全自动，基于双方属性计算 |
| 单场时间 | ≤30 秒（文字播报） |

### 3.2 对手生成

```
opponent_power = player_power × (0.8 + random() × 0.4)  // ±20%
opponent_atk = player_atk × (0.8 + random() × 0.4)
opponent_def = player_def × (0.8 + random() × 0.4)
opponent_hp  = player_hp × (0.8 + random() × 0.4)
```

对手名字从 NPC 名池随机抽取，附带境界和仙盟名。

### 3.3 战斗计算

复用现有 BattleCalc 的伤害/防御公式，双方交替攻击：

```
回合制（最多 20 回合）：
1. 玩家攻击 → 敌方受伤
2. 敌方攻击 → 玩家受伤
3. 任一方 HP ≤ 0 → 结束
4. 20 回合未分胜负 → HP% 高者胜
```

神通被动加成生效，主动技能不生效（异步 PvP）。
天赋/灵兽/策略加成均生效。

### 3.4 仙誉积分

| 结果 | 仙誉变化 | 备注 |
|------|---------|------|
| 胜利 | +15~25（对手越强越多） | base=15, +(opponent_power/player_power-0.8)×50 |
| 失败 | -5 | 固定 |
| 平局 | +5 | 罕见 |

每周一 00:00 结算排名，发放奖励后重置为赛季基础分（当前分 ×0.5）。

### 3.5 赛季排名奖励（每周结算）

| 排名 | 灵石 | 金币 | 特殊 |
|------|------|------|------|
| 第 1~10 | 100 | 50,000 | 称号「论道宗师」(攻击+5%) |
| 第 11~50 | 50 | 25,000 | 强化符 ×5 |
| 第 51~100 | 20 | 10,000 | — |

称号持续一周，下次结算刷新。

### 3.6 仙誉兑换商店

| 物品 | 仙誉费用 | 限购 |
|------|---------|------|
| 高级修为丹(修为+8小时) | 100 | 每周 3 |
| 灵兽突破丹 | 150 | 每周 2 |
| 天赋果(天赋点+1) | 300 | 每周 1 |
| 称号「战神」(暴击+3%) | 500 | 永久 1 次 |
| 神通秘籍(悟道值+300) | 400 | 每周 1 |

### 3.7 战斗日志

每场战斗生成文字日志，可回看：


## 4. 仙界盛会（限时活动模板）

### 4.1 活动框架

在现有 `event.ts` 基础上新增 3 个活动模板。活动由系统按排程触发。

| 属性 | 值 |
|------|-----|
| 活动频率 | 每周 2~3 次（随机类型） |
| 活动时长 | 1~4 小时（按类型） |
| 同时进行 | 最多 1 个活动 |

### 4.2 修炼竞速

| 属性 | 值 |
|------|-----|
| 时长 | 2 小时 |
| 目标 | 限时内修炼次数排名 |
| 参与者 | 玩家 + 19 个 NPC |
| NPC 模拟 | 速率 = 玩家速率 × (0.5~1.5) 随机 |

**奖励**：

| 排名 | 奖励 |
|------|------|
| 第 1 | 灵石 ×30 + 修为丹(大) ×3 |
| 第 2~5 | 灵石 ×15 + 修为丹(中) ×3 |
| 第 6~10 | 灵石 ×5 + 修为丹(小) ×5 |
| 参与奖 | 金币 ×5,000 |

### 4.3 Boss 共伐

| 属性 | 值 |
|------|-----|
| 时长 | 4 小时 |
| 目标 | "全服"累计伤害击杀限时 Boss |
| Boss HP | 玩家战力 × 500 |
| 玩家伤害 | 每次攻击 = 正常战斗伤害 × 0.1 |
| NPC 贡献 | Boss HP 的 40%~60%，按时间线性增长 |

**机制**：
- 玩家点击攻击 Boss（不限次数，每次有 3s 冷却）
- 击杀成功：全员奖励
- 未击杀：参与奖

**奖励**：

| 条件 | 奖励 |
|------|------|
| 击杀成功 — 伤害第 1 | 灵石 ×50 + 灵兽丹 ×3 |
| 击杀成功 — 前 30% | 灵石 ×25 + 强化符 ×3 |
| 击杀成功 — 参与 | 灵石 ×10 + 金币 ×10,000 |
| 未击杀 — 参与 | 金币 ×5,000 |

### 4.4 寻宝夺旗

| 属性 | 值 |
|------|-----|
| 时长 | 1 小时 |
| 机制 | 每 5 分钟随机弹出 1 个宝箱事件 |
| 宝箱总数 | 12 个 |
| 每个宝箱 | 限时 30 秒点击领取 |

**宝箱奖励池**（随机）：

| 等级 | 概率 | 奖励范围 |
|------|------|---------|
| 🟤 铜宝箱 | 50% | 金币 1,000~5,000 |
| ⚪ 银宝箱 | 30% | 灵石 3~8 + 金币 3,000 |
| 🟡 金宝箱 | 15% | 灵石 10~20 + 强化符 ×1 |
| 🔴 仙宝箱 | 5% | 灵石 30 + 灵兽丹 ×2 + 天赋果 ×1 |

> NPC 会"抢"走玩家未及时点击的宝箱（30s 后自动消失）。

### 4.5 活动排程


## 8. 经济影响评估

### 8.1 新增资源产出/日

| 来源 | 灵石 | 金币 | 贡献点 | 仙誉 |
|------|------|------|--------|------|
| 仙盟任务(3个全完) | — | 18,000 | 140 | — |
| 仙盟仓库领取 | — | — | -140 | — |
| PvP(5胜) | — | — | — | 100 |
| PvP 周结算(avg #30) | ~7/日 | ~3,500/日 | — | — |
| 活动(avg 2次/周) | ~5/日 | ~3,000/日 | — | — |
| **净增** | **~12** | **~24,500** | **0 (自循环)** | **100** |

### 8.2 评估

- 灵石日增 +12：占中期收入的 ~15%，可接受
- 金币日增 +24,500：占中期收入的 ~80%，**偏高**
  - **建议**：仙盟任务金币降至 3,000/个，活动参与奖降至 3,000
  - 调整后金币日增 ≈ +12,000（~40%），合理
- 贡献点自循环（捐→领），不影响外部经济 ✅
- 仙誉独立货币，不可与灵石/金币互换 ✅

## 11. 技术注意事项

1. **NPC 模拟一致性**：NPC 成员/对手数据基于玩家实力动态生成，需 seed 保证同日内稳定
2. **PvP 战斗引擎**：复用 BattleCalc，新增回合制交替攻击模式（vs 现有连续攻击）
3. **活动调度器**：基于 `weeklySchedule` + 系统时间判断，tick 中检查
4. **仙盟任务 NPC 贡献**：在 tick 中模拟 NPC 进度增长，避免瞬间完成
5. **存档兼容**：旧存档无 alliance/pvp/festival 字段时初始化默认（未加入仙盟、仙誉0、无活动）
6. **经济调整**：本 PRD 建议调低仙盟任务金币和活动参与奖（见 §8.2）
