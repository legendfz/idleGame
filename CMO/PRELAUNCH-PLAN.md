## 1. 目标用户画像细化

基于 MARKET-PLAN.md 的三层用户模型，进一步细化为可执行的触达对象。

### 1.1 种子用户（首批 500 人）

| 标签 | 画像 | 在哪里 | 触达方式 |
|------|------|--------|---------|
| 🎮 Idle 硬核玩家 | 玩过 5+ 款 idle game，关注数值平衡 | r/incremental_games, idle 类 Discord | 开发日志 + 内测邀请 |
| 💻 独立游戏开发者 | 自己做过或关注独立游戏 | V2EX, IndieNova, GameJam 社区 | 技术分享 + 互推 |
| 📖 文字游戏爱好者 | 玩过 A Dark Room / Candy Box | 同上 + HN / 少数派 | 「纯文字新作」话题引入 |

### 1.2 早期采纳者（500→5,000 人）

| 标签 | 画像 | 在哪里 | 触达方式 |
|------|------|--------|---------|
| 🐒 西游文化粉 | 西游迷/国漫粉/悟空IP爱好者 | B站、微博、小红书 | 西游梗内容 + 游戏预告 |
| 📱 轻度放置玩家 | 日常挂机玩家，不想肝 | TapTap、App Store 浏览 | 商店页面优化 + 预约 |
| 🔤 极简/复古审美 | 喜欢 terminal 风格、像素风 | 即刻、推特、设计社区 | 视觉风格展示 |

### 1.3 用户行为假设

```
典型使用场景：
- 早上通勤：打开看离线收益 → 点几下升级 → 关掉（2分钟）
- 午休：推进一段剧情 → 安排挂机策略（5分钟）
- 睡前：看看今天收了多少经验值 → 设置过夜策略（3分钟）
```

## 2. 社区建设策略

### 2.1 社区矩阵

| 平台 | 定位 | 目标 | 启动时间 |
|------|------|------|---------|
| **QQ 群** | 核心玩家据点，内测反馈 | 种子用户聚集 | MVP 内测前 2 周 |
| **Discord** | 海外社区 + 开发透明 | 英文用户 + 开发日志 | MVP 内测同步 |
| **TapTap 论坛** | 官方互动，版本公告 | 预约转化 | 预热启动即刻 |
| **微信群** | 高粘性小圈子（限 200 人） | VIP 内测用户 | 内测启动 |

### 2.2 社区内容规划

**开发日志系列**（每周 1 篇，预热期共 6-8 篇）

| 期数 | 主题 | 平台 |
|------|------|------|
| #1 | 「为什么我要用符号做一个西游记」— 创意起源 | V2EX/少数派/Reddit |
| #2 | 「🐵→🦍→👑 从石猴到齐天大圣的符号进化」— 角色设计 | 全平台 |
| #3 | 「如何用 ⚔️💥🔥 表现七十二变」— 战斗系统 | Reddit/V2EX |
| #4 | 「离线八小时，悟空替你打了 1,000 只小妖」— Idle 机制 | 全平台 |
| #5 | 「五行山下压了 500 年，存档读取的另一种说法」— 彩蛋 | 全平台 |
| #6 | 「内测数据：玩家平均第几天大闹天宫？」— 数据故事 | 全平台 |

### 2.3 玩家共创

- **符号投票**：让社区投票选择新妖怪/法宝的符号表示
- **取名大赛**：为游戏中的法宝/技能征集名称
- **Bug 猎人计划**：内测 Bug 反馈者获得独占称号

## 3. 首发渠道选择

### 3.1 优先级排序

| 优先级 | 渠道 | 理由 | 首发动作 |
|--------|------|------|---------|
| 🥇 P0 | **TapTap** | 独立游戏氛围最好，预约→下载转化高 | 开预约页，定期更新开发动态 |
| 🥇 P0 | **r/incremental_games** | 文字 idle 游戏的精准社区 | 开发日志系列帖 |
| 🥈 P1 | **V2EX / 少数派** | 技术+独立游戏交叉用户 | 创意文章 + 内测邀请 |
| 🥈 P1 | **IndieNova** | 国内独立游戏社区 | 游戏页面 + 开发日志 |
| 🥉 P2 | **B 站** | 视频展示「纯文字打 boss」反差感 | 1-2 条短视频 |
| 🥉 P2 | **小红书** | 「极简游戏美学」话题 | 截图+安利帖 |

### 3.2 暂不投入

| 渠道 | 理由 | 何时启动 |
|------|------|---------|
| 抖音 | 需要视频制作能力，预热期资源有限 | 正式上线后 |
| 微信公众号 | 冷启动慢，等有用户基础再做 | 正式上线后 |
| Product Hunt / HN | 英文版未就绪 | 海外版本上线时 |

## 4. 预热时间线

假设 MVP 开发周期 6-8 周，以下为预热营销配合节奏。

### Phase 0：准备期（第 1-2 周）
> MVP 开发启动，营销筹备

| 时间 | 动作 | 负责 | 交付物 |
|------|------|------|--------|
| W1 | 确定游戏名称和 Slogan | CMO | 命名方案 |
| W1 | 注册 TapTap / IndieNova 开发者账号 | CMO | 账号就绪 |
| W1 | 创建 QQ 群 / Discord 服务器 | CMO | 社区上线 |
| W2 | 制作 TapTap 预约页（纯文字截图 + 描述） | CMO | 预约页上线 |
| W2 | 撰写开发日志 #1 | CMO | 文章发布 |

### Phase 1：造势期（第 3-5 周）
> MVP 开发中，持续内容输出

| 时间 | 动作 | 交付物 |
|------|------|--------|
| W3 | 开发日志 #2 — 角色符号设计 | 文章 + 社区讨论 |
| W3 | Reddit r/incremental_games 首帖 | 社区帖子 |
| W4 | 开发日志 #3 — 战斗系统 | 文章 |
| W4 | V2EX/少数派 创意文章 | 文章 |
| W5 | 开发日志 #4 — Idle 机制 | 文章 |
| W5 | 符号投票活动（社区互动） | 投票帖 |

**里程碑**：TapTap 预约 1,000+

### Phase 2：内测期（第 6-7 周）
> MVP 可玩，邀请种子用户

| 时间 | 动作 | 交付物 |
|------|------|--------|
| W6 | 封闭内测启动（100 人） | 内测版本分发 |
| W6 | 内测反馈收集（QQ群/Discord） | 反馈汇总 |
| W6 | 开发日志 #5 — 彩蛋故事 | 文章 |

| W7 | 开发日志 #6 — 内测数据 | 文章 |

**里程碑**：TapTap 预约 3,000+，内测次留 >35%

### Phase 3：上线冲刺（第 8 周）
> 正式发布

| 时间 | 动作 | 交付物 |
|------|------|--------|
| W8 前半 | 上线倒计时内容（「还有 3 天到花果山」） | 社交媒体系列 |
| W8 前半 | 预热所有渠道同步推送 | 推送通知 |
| W8 | **正式上线** | 全渠道发布 |
| W8 后半 | 上线日 Reddit/V2EX/TapTap 同步帖 | 帖子发布 |
| W8 后半 | 社区活动：首周成就挑战赛 | 活动上线 |

**里程碑**：首周下载 10,000+

## 5. 预算估算

### 预热期总预算：¥5,000 - ¥10,000

| 项目 | 明细 | 预算 |
|------|------|------|
| **社区运营** | QQ群/Discord管理 | ¥0（自运营） |
| **内容创作** | 开发日志撰写 | ¥0（自产出） |
| **平台费用** | TapTap/IndieNova 开发者注册 | ¥0（免费） |
| **B站短视频** | 1-2条录屏+简单剪辑 | ¥0-500（自制）或 ¥1,000-2,000（外包） |
| **社区激励** | 内测用户奖励/Bug猎人 | ¥1,000-2,000（虚拟称号为主） |
| **应急储备** | KOL合作/突发需求 | ¥2,000-5,000 |
| **域名/官网** | 简单落地页 | ¥500-1,000 |

### 预算说明

- **核心策略是零成本口碑传播**，预算主要用于锦上添花
- 如果开发日志自然传播效果好，可将应急储备投入 B 站 UP 主合作
- 正式上线后的推广预算另行规划

### 成本效率目标

| 指标 | 目标 |
|------|------|
| 预热期 CPA（每预约成本） | < ¥2 |
| 内测用户获取成本 | ≈ ¥0（纯社区邀请） |
| 首周下载 CPI | < ¥1 |

## 6. 关键风险与应对

| 风险 | 概率 | 应对 |
|------|------|------|
| 社区帖子无人关注 | 中 | 多平台分发，A/B测试标题 |
| 内测留存低 | 中 | 快速迭代，优先修核心循环 |
| MVP 延期 | 中 | 预热节奏灵活调整，多备内容 |
| 竞品模仿 | 低 | 先发优势 + 社区壁垒 |
