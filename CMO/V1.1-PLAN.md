## Part 2：用户反馈渠道搭建方案

### 2.1 反馈渠道矩阵

| 渠道 | 类型 | 适合收集 | 优先级 |
|------|------|---------|--------|
| **GitHub Issues** | 结构化 | Bug 报告、功能请求 | ⭐⭐⭐ |
| **Discord #feedback** | 实时讨论 | 体验反馈、建议 | ⭐⭐⭐ |
| **QQ 群** | 实时讨论 | 中文用户反馈 | ⭐⭐⭐ |
| **TapTap 评论区** | 公开评价 | 用户口碑、评分 | ⭐⭐ |
| **Reddit 帖评论** | 社区讨论 | 深度反馈 | ⭐⭐ |
| **游戏内反馈入口** | 低门槛 | 即时反馈 | ⭐⭐ |

### 2.2 GitHub Issues 模板

建议在 GitHub 仓库创建 Issue Templates：

**Bug Report 模板**：
```markdown
---
name: Bug Report 🐛
about: 发现了问题？告诉我们
---

**描述**：
**复现步骤**：1. 2. 3.
**预期行为**：
**实际行为**：
**设备/浏览器**：
**截图**（如有）：
```

**Feature Request 模板**：
```markdown
---
name: Feature Request ✨
about: 有想法？我们想听
---

**功能描述**：
**为什么需要**：
**可能的实现方式**（可选）：
```

### 2.3 Discord 服务器架构

```
西游 Idle Discord
├── 📢 announcements     — 版本更新、活动公告（只读）
├── 💬 general           — 闲聊
├── 🐛 bug-reports       — Bug 反馈（用 Forum 频道）
├── 💡 suggestions       — 功能建议（用 Forum 频道）
├── 📊 polls             — 投票决策
├── 🏆 achievements      — 晒成就/进度
└── 📝 dev-log           — 开发日志同步
```

### 2.4 游戏内反馈入口（需 CTO 配合）

在游戏设置/关于页面加一个简单反馈入口：

```
📬 反馈
├── 🐛 报告 Bug → 跳转 GitHub Issue（预填模板）
├── 💡 建议功能 → 跳转 GitHub Issue
├── 💬 加入社区 → Discord / QQ 群链接
└── ⭐ 给个好评 → TapTap 评价页
```

### 2.5 反馈处理流程

```
用户反馈
  ↓
收集（各渠道汇总到统一看板）
  ↓
分类（Bug / 功能请求 / 体验问题 / 赞美）
  ↓
优先级排序（频率 × 影响度）
  ↓
响应（24h 内回复用户）
  ↓
跟进（纳入版本计划 / 标记 won't fix + 说明原因）
  ↓
闭环（修复后通知原反馈者）
```

### 2.6 反馈运营规范

| 规则 | 说明 |
|------|------|
| 24h 响应 | 所有反馈 24 小时内给出首次回复 |
| 感谢优先 | 先感谢，再讨论方案 |
| 透明沟通 | 不做的功能也要解释为什么 |
| 归功用户 | changelog 中注明「感谢 @xxx 反馈」 |
| 周报汇总 | 每周汇总反馈趋势，同步给 CPO/CTO |
