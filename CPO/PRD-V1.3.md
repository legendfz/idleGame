## 1. 副本/关卡系统（西游取经路线 Boss 战）

### 1.1 设计理念

以西游取经路线为主线，每个地点对应一个副本关卡。玩家需达到指定等级/境界方可挑战，击败 Boss 获得丰厚奖励和剧情推进。

### 1.2 取经路线地图

| 章节 | 地点 | Boss | 推荐等级 | 境界要求 |
|------|------|------|---------|---------|
| 第一难 | 🏔️ 五行山 | 五行山石灵 | Lv.10 | 通灵 |
| 第二难 | 🌊 鹰愁涧 | 小白龙（加入队友） | Lv.20 | 通灵 |

| 第五难 | 🔥 火焰山 | 🐉 牛魔王 & 铁扇公主 | Lv.70 | 金丹 |
| 第六难 | 🕸️ 盘丝洞 | 🕷️ 七个蜘蛛精 | Lv.85 | 元婴 |

### 1.3 副本机制

**进入条件**：
- 达到推荐等级和境界要求
- 前置副本已通关（线性解锁）
- 每个副本每日可挑战 3 次（首通无限制）

**战斗规则**：
- 副本分 3 个阶段：清小怪（5 波）→ 精英怪（2 波）→ Boss 战
- Boss 有专属技能和阶段机制（血量低于 50% 进入狂暴）
- 战斗时间限制：300 秒，超时判定失败
- 队友参战：已收服的队友自动参战，提供额外输出

**Boss 技能示例（牛魔王）**：
```
═══ 🔥 火焰山 · Boss 战 ═══
🐉 牛魔王  ██████████  HP: 580,000/580,000

⚠️ 技能预告：[铁扇烈焰] 3 秒后释放
💥 全体伤害 -12,000
🛡️ 提示：点击「闪避」减少 50% 伤害

📊 阶段：1/2  |  ⏱️ 剩余：287秒
```

**Boss 阶段机制**：
- 阶段 1（100%~50% HP）：正常攻击 + 单体技能
- 阶段 2（50% 以下）：狂暴状态，攻击力 +50%，释放全体技能

```typescript

}
```

## 2. 成就系统

### 2.1 设计理念

成就分两类：**里程碑成就**（自然达成）和 **挑战成就**（主动挑战）。成就提供永久属性加成和展示荣誉。

| 🐒 灵猴初醒 | 角色达到 Lv.10 | 攻击 +5% |

| 👥 师徒齐聚 | 收服所有队友（4 人） | 队友攻击 +10% |

## 3. 排行榜系统

### 3.1 设计理念

本地排行榜为核心，记录玩家自己的历史最佳成绩。预留在线排行榜接口，后续版本可接入服务端。

### 3.2 排行榜维度

| 排行榜 | 排序依据 | 说明 |
|--------|---------|------|
| 🏆 战力排行 | 综合战斗力 | 攻击+防御+速度+装备的加权值 |
| ⚔️ 副本速通 | 通关时间 | 每个副本独立计时排行 |
| 💀 击杀排行 | 累计击杀数 | 总击杀怪物数量 |
| 📈 等级排行 | 角色等级 | 最高等级 |
| 💎 收集排行 | 装备收集数 | 拥有不同装备种类数 |

### 3.3 本地排行实现

```

【🏆 历史战力峰值】

【💀 累计击杀】
```

每个排行榜本地保存 Top 10 记录，含时间戳。

### 3.4 在线排行预留

```typescript
interface LeaderboardEntry {
  playerId: string;          // 本地用 'local'，在线用服务端 ID
  playerName: string;
  score: number;
  timestamp: number;
  metadata?: Record<string, any>;  // 副本名、装备信息等
}

interface LeaderboardAPI {
  // 本地实现
  getLocalRanking(type: string): LeaderboardEntry[];
  submitLocalScore(type: string, entry: LeaderboardEntry): void;

  // 在线预留（v1.3 不实现，仅定义接口）
  getOnlineRanking?(type: string, page: number): Promise<LeaderboardEntry[]>;
  submitOnlineScore?(type: string, entry: LeaderboardEntry): Promise<void>;
}
```

### 3.5 战力计算公式

```
战斗力 = (攻击 × 1.0 + 防御 × 0.8 + 速度 × 0.5 + HP × 0.3)
       × (1 + 装备总倍率 × 0.01)
       × (1 + 成就加成)
       × 境界系数
```

境界系数：灵猴初醒 1.0 / 通灵 1.1 / 炼气 1.3 / 筑基 1.6 / 金丹 2.0 / 元婴 2.5 / 化神 3.0 / 大乘 4.0

## 4. 版本总结

### 4.1 开发优先级

| 优先级 | 系统 | 工作量预估 |
|--------|------|-----------|
| P0 | 副本/关卡系统 | 大（核心玩法） |
| P1 | 成就系统 | 中（数据追踪+UI） |
| P2 | 排行榜系统 | 小（本地存储+展示） |

### 4.2 依赖关系

- 成就系统依赖副本系统（副本相关成就）
- 排行榜依赖副本系统（速通排行）和战力计算
- 建议开发顺序：副本 → 成就 → 排行榜
