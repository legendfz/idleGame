# PRD：背包溢出智能分解

**版本**：v1.2 Sprint 2
**日期**：2026-03-01
**作者**：CPO

---

## 1. 概述

当前背包上限 50，满时装备直接丢失。本功能增加满包提示、智能推荐分解、批量操作，并与精炼系统联动产出精炼材料。

---

## 2. 分解产出

### 2.1 产出表

| 品质 | 灵石返还 | 仙石 | 精炼碎片 | 说明 |
|------|---------|------|---------|------|
| ⬜ 凡品 | baseStat × 5 | 0 | 0 | |
| 🟢 良品 | baseStat × 10 | 5 | 0 | |
| 🔵 上品 | baseStat × 25 | 20 | 0 | |
| 🟣 极品 | baseStat × 60 | 50 | 0 | |
| ✦ 仙品 | baseStat × 150 | 100 | 0 | |
| 🟡 混沌 | baseStat × 300 | 200 | 0 | |

### 2.2 与精炼系统联动
- 分解🔴鸿蒙装备获得鸿蒙碎片 ×2（高于精炼失败的 ×1）
- 设计意图：鼓励玩家分解多余鸿蒙而非囤积

## 3. 背包满提示

### 3.1 触发条件
- 背包 ≥ 45/50 时：顶部出现黄色提示条 `⚠️ 背包即将满 (47/50)`
- 背包 = 50/50 时：红色提示条 `🚫 背包已满！新掉落将自动分解`

### 3.2 自动分解（离线）
离线掉落超出背包时自动分解（沿用 v1.1 机制）：
- 优先分解低品质
- 分解排序：凡品 → 良品 → 上品 → …（同品质按 baseStat 升序）
- 产出汇入离线灵石/仙石

### 3.3 在线掉落满包
在线战斗掉落时背包已满：
- 弹出分解确认弹窗（不自动分解）
- 弹窗内容：「背包已满，是否分解新掉落？」
- 选项：[分解获得 🪙X + 💎Y] [打开背包管理]

## 4. 智能推荐分解

### 4.1 推荐算法

```
1. 排除已装备物品
2. 排除已锁定物品（新功能：长按锁定）
3. 按综合评分升序排列（低分优先推荐分解）
4. 评分 = 品质权重 × 100 + 有效属性 + (强化等级 × 50) + (有被动 ? 200 : 0) + (套装件 ? 300 : 0)
```

品质权重：凡=1, 良=2, 上=3, 极=4, 仙=5, 混沌=6, 鸿蒙=7

### 4.2 推荐 UI

```
═══ 🔧 智能分解推荐 ═══
推荐分解以下低价值装备：

☑️ ⬜ 木棍 +0       → 🪙40
☑️ ⬜ 猴皮甲 +0     → 🪙150
☑️ 🟢 铁棒 +2       → 🪙380 + 💎5
☐ 🔵 龙鳞铠 +0      → 🪙10,000 + 💎20

合计回收：🪙570 + 💎5
已选 3 件 / 可选 48 件

[♻️ 分解所选]  [全选凡品]  [取消]
```

## 5. 批量分解

### 5.1 操作方式
- 背包页新增「♻️ 批量分解」按钮
- 进入分解模式：物品可多选（勾选框）
- 快捷按钮：[全选凡品] [全选良品] [全选上品以下] [智能推荐]

### 5.2 安全机制
- 极品(★)及以上品质分解需**二次确认**：「确定分解 1 件极品装备？」
- 已装备物品不可选中
- 锁定物品不可选中（显示 🔒）
- 分解确认弹窗显示总回收预览

### 5.3 装备锁定
- 背包中长按装备 → 锁定/解锁切换
- 锁定的装备左上角显示 🔒
- 锁定装备不可分解、不可出售、不可作为精炼材料
- 锁定状态持久化到存档

## 7. 用户故事

### US-BO01: 背包满提示
> 作为玩家，背包快满时我想收到提醒。

验收标准：
- [ ] 45/50 时黄色提示，50/50 时红色提示
- [ ] 在线满包掉落弹确认框（不自动丢弃）

### US-BO02: 智能推荐分解
> 作为玩家，我想系统推荐低价值装备供我分解。

验收标准：
- [ ] 按评分升序推荐，排除已装备/已锁定
- [ ] 显示每件回收预览（灵石+仙石）
- [ ] 可全选/取消

### US-BO03: 批量分解
> 作为玩家，我想一次分解多件装备。

验收标准：
- [ ] 背包页「♻️ 批量分解」进入多选模式
- [ ] 快捷按钮：全选凡品/良品/上品以下/智能推荐
- [ ] 极品+分解需二次确认
- [ ] 确认后一次性获得所有回收材料

### US-BO04: 装备锁定
> 作为玩家，我想锁定重要装备防止误分解。

验收标准：
- [ ] 长按装备切换锁定
- [ ] 🔒 标记显示
- [ ] 锁定装备不可分解/出售/作为精炼材料
- [ ] 锁定状态持久化
