# PRD：离线掉落保底机制

**版本**：v1.2 Sprint 2
**日期**：2026-03-01
**作者**：CPO

---

## 1. 概述

离线挂机装备掉落纯靠概率，长时间离线可能全是凡品。保底机制确保玩家离线回归有确定性奖励，减少"白挂"挫败感。

---

## 2. 保底规则

### 2.1 触发阈值

| 阈值 | 离线关卡数 | 约等于时长 | 保底掉落 |
|------|-----------|-----------|---------|
| 银保底 | 100 关 | ~2.5 小时 | 至少 1 件 🟢良品+ |
| 金保底 | 500 关 | ~12.5 小时 | 至少 1 件 🔵上品+ |

关卡数计算 = `floor(离线秒数 / 90)`（每关 90 秒预估）。

### 2.2 触发逻辑

在离线结算时，按以下顺序执行：

```
1. 正常 roll 所有 Boss 掉落（现有逻辑）
2. 统计已掉落装备的最高品质
3. 如果 关卡数 ≥ 500 且 无🔵上品+  → 从上品+池中额外 roll 1 件
4. 如果 关卡数 ≥ 100 且 无🟢良品+  → 从良品+池中额外 roll 1 件
5. 保底装备追加到掉落列表末尾
```

### 2.3 保底掉落池

**银保底（良品+）**：

| 品质 | 权重 |
|------|------|
| 🟢 良品 | 70 |
| 🔵 上品 | 25 |
| 🟣 极品 | 4.5 |
| ✦ 仙品 | 0.5 |

**金保底（上品+）**：

| 品质 | 权重 |
|------|------|
| 🔵 上品 | 60 |
| 🟣 极品 | 30 |
| ✦ 仙品 | 9 |
| 🟡 混沌 | 1 |

装备从当前关卡可掉落池中按权重随机。

### 2.4 多次保底

离线 1000 关 = 触发 2 次银保底 + 2 次金保底？**不**。

保底**不叠加**，每类最多触发 1 次。原因：
- 避免长时间离线收益过于爆炸
- 保底是"兜底"而非"奖励倍增"

### 2.5 与现有系统整合

| 现有逻辑 | 变更 |
|---------|------|
| `load()` 中离线计算 | 在正常 roll 后追加保底检查 |
| `offlineReport` | 增加 `guaranteeDrops: EquipmentItem[]` 字段 |
| 离线报告 UI | 保底装备单独标记 `🎁 保底奖励` |

## 4. 数值验证

| 场景 | 正常掉落期望 | 保底补充 | 合理性 |
|------|------------|---------|--------|
| 离线 2.5h (100关) | ~30 件，大部分凡品 | 保底 1 件良品+ | ✅ 兜底不过度 |
| 离线 8h (320关) | ~96 件，期望有良品 | 多数情况不触发银保底 | ✅ 保底少触发 |
| 离线 12.5h (500关) | ~150 件，期望有上品 | 金保底兜底 | ✅ 安全网 |
| 离线 24h (960关) | ~288 件 | 金+银各触发 1 次 | ✅ 不叠加 |

## 6. 用户故事

### US-OG01: 银保底
> 作为玩家，离线 100 关以上回来，我至少能获得 1 件良品装备。

验收标准：
- [ ] 离线 ≥100 关且正常掉落无良品+ → 额外获得 1 件良品+
- [ ] 已有良品+时不触发
- [ ] 离线报告标记「🎁 保底奖励」

### US-OG02: 金保底
> 作为玩家，离线 500 关以上回来，我至少能获得 1 件上品装备。

验收标准：
- [ ] 离线 ≥500 关且正常掉落无上品+ → 额外获得 1 件上品+
- [ ] 保底与银保底独立，可同时触发
- [ ] 保底不叠加（超 1000 关仍只触发 1 次金保底）
