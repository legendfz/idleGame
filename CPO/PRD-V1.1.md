|---------|---------|-------|---------|

## 2. 离线挂机装备掉落

### 2.1 机制设计
v1.0 离线仅产出灵石 + 经验。v1.1 增加装备掉落。

**离线装备掉落规则**：
- 基于离线击杀的怪物数量计算掉落
- 离线击杀数 = `floor(离线秒数 / 每关预估秒数)`
- 每关预估秒数 = 90 秒（取「刚好达标」中位数）
- 每只 Boss（每关 1 只）按 Boss 掉落概率 roll 一次
- 小怪不产出装备（离线简化，避免背包爆满）

### 2.2 离线装备掉落数学

| 离线时长 | 预估关卡数 | 预估 Boss 击杀 | 期望装备数 |
|---------|-----------|---------------|-----------|
| 1 小时 | 40 | 40 | ~12（Boss 30% 掉率） |
| 4 小时 | 160 | 160 | ~48 |
| 8 小时 | 320 | 320 | ~96 |
| 24 小时 | 960 | 960 | ~288 |

**问题**：长时间离线掉落过多，背包爆满。

### 2.3 背包溢出保护
- 离线掉落上限 = `min(掉落数, 背包剩余空间)`
- 超出部分自动分解为灵石（按出售价计算）
- 离线报告显示：`📦 获得装备 X 件（Y 件因背包已满自动分解为 🪙Z）`

### 2.4 离线掉落品质调整
离线掉落品质概率与在线 Boss 一致，但增加保底：
- 每 100 关离线保底 1 件 🔵仙品+
- 每 500 关离线保底 1 件 🟣神品+

## 3. 游戏内反馈入口

### 3.1 设计方案
基于 FEEDBACK-PLAN.md，实现最小可用版本。

### 3.2 UI 设计

设置页（更多 Tab）新增区块：

```
═══ 📝 反馈与帮助 ═══
[🐛 报告 Bug]  [💡 功能建议]  [⚖️ 数值反馈]
```

三个按钮分别打开不同的 GitHub Issue 预填链接：
- Bug：`/issues/new?template=bug.yml&title=[Bug]&labels=bug`
- 功能：`/issues/new?template=feedback.yml&title=[建议]&labels=enhancement`
- 数值：`/issues/new?template=feedback.yml&title=[数值]&labels=balance`

### 3.3 自动附带游戏信息
链接自动附带当前游戏状态作为 URL 参数（编码到 body）：

```
Lv.{level} | {realmName} | 第{chapter}章-{stage}关 | 游戏时间{totalPlayTime}
```

帮助开发者快速定位问题。

### 3.4 代码变更范围
- `App.tsx` → `SettingsView`：增加反馈区块 + 3 个按钮
- 新增 `utils/feedback.ts`：生成预填 Issue URL 的工具函数
- 仓库根目录：`.github/ISSUE_TEMPLATE/feedback.yml` + `bug.yml`

### 3.5 用户故事

**US-22: 游戏内反馈按钮**
> 作为玩家，我想在设置页快速提交反馈，以便报告问题或提建议。

验收标准：
- [ ] 设置页显示 3 个反馈按钮（Bug / 功能 / 数值）
- [ ] 点击按钮打开新标签页到 GitHub Issue 页面
- [ ] Issue 自动填入模板 + 游戏状态信息
- [ ] 无 GitHub 账号时页面正常显示（GitHub 登录页）
